What's New in Version 3.0

Offline work


Works when the Internet is not

Using TeamDrive to turn the Internet into a virtual hard drive offers many advantages. The three most frequently quoted are: Being able to access files from different locations, Sharing files with other users and Eliminating the need for subsequent backups. When the Internet crashes, these files normally are no longer accessible. With TeamDrive's Offline feature, any files recently accessed or marked as 'Available Offline' will still be accessible even if and when the Internet is not.

Synchronization

When Internet access is available again, the files are synchronized back to their original locations.
 

SSL secure connections

Protects data

When data is traveling over the Internet, it is possible that it may be read by unauthorized readers. With SSL secure connections, all data is encrypted as it travels over the Internet, ensuring that it is protected and available only to authorized users.
 
TeamLinks


Share files easily

The easiest way to share files with others is with TeamDrive's TeamLink feature. TeamLinks enables users to link to and work on original documents, rather than emailing documents to one another. TeamLinks eliminates version control issues and can be easily inserted into email, web pages, manual, etc. Then, when a user clicks on the TeamLink, the file is automatically opened and ready for collaborative editing. If TeamDrive is not installed on user's drive, it will automatically self-install at that time.

Improve access to documents

The most popular way that people currently share documents is by emailing versions to one another. The majority of computer users know how to send e-mails and also add attachments. This has the advantage of being easy to do but has several disadvantages as well:

Disadvantages of email attachments for collaboration

  1. Sending large attachments can take a long time, especially when using dialup connections.
  2. When one user updates a document, it needs to be re-sent to all recipients.
  3. When a recipient updates the document, it needs to be sent back to the sender.
  4. When more than one recipient updates a document, all changes will have to be manually merged back into the original document.
  5. E-mail is not secure. Other people may be able to read the attachment.

Provides collaborative editing and instant updates

Any changes made to documents are stored directly in the original document; there is no need to email attachments back-and-forth. The next time the user clicks on the TeamLink, the most current version of the document is made available to that user.

Documents are always current

Whenever collaborators click on TeamLink, they will always see a current copy of the document. This is useful for sales and marketing information, technical manuals and other documents where the currency of information is critical.

Team editing

TeamLinks makes it easy for two or more collaborators to edit a document. Instead of e-mailing changes back and forth and trying to determine who has the most current copy, all team members edit the same copy of the document. When one person is editing the document, it is 'locked', indicating that an edit is in process. Other team members can subsequently make their edits once the document is unlocked.

Secure transmission

All attachments are sent securely. When used with the TeamDrive server, or other secure servers, TeamDrive will use SSL to send the document to ensure that unauthorized users will not be able to access and read the attachments.

"Whoops" control

TeamLink emails also provide a form of "whoops" control so that if users mistakenly send a document through a TeamLink email, then decide to change or delete that document, they can do so even after the email has been sent.

Access control

TeamLinked documents can be either read-only or read-write.

Faster sends

Instead of waiting for entire documents to be sent via email, only the short TeamLink is sent. The document is then uploaded in the background.
 
Time zone support


Ease of use across multiple time zones

When team members are working in different time zones it is often difficult to know the time and date that a file was last edited; confusion can also arise over which file or which version of a file is the most current. With time zone support, all file times are converted to the local time of the person accessing the file, eliminating any confusion.
 

Auto proxy detection


Easy setup with firewalls

Many computer users employ proxies in order to share an Internet connection or to shield them from possible hacker attacks. In the past, users would need to know details of the proxy support in order to configure TeamDrive. With auto proxy detection, TeamDrive detects proxies and configures itself automatically in most cases.
 

Cancel file transfers


Eliminates wait on mistaken downloads

Users sometimes mistakenly try to open a large file, then realize that it will take some time for TeamDrive to retrieve it, especially with slow Internet connections. With this new feature, users can cancel file transfers when their time is limited.

TeamDrive Key Features

Drive-letter mapping

Turns the Internet into a virtual drive

The TeamDrive client maps a drive-letter to an Internet server, effectively turning the Internet into a secure, shared virtual hard drive. TeamDrive allows any standard Windows program to directly open and edit any document as easily as if it were on the user's own computer.
 
In-place editing

Edit like any other document

Some remote document storage systems make users download a document to their local drive, edit it there and then upload it when finished. With TeamDrive, this is unnecessary. Users just use the File/Open menu on any application to directly open the document and then save it as usual.
 
Instant saves

Resume working immediately

When a file is saved, TeamDrive caches it locally and then allows work to resume. In the background, TeamDrive is uploading that file to the server so the user never needs to wait for the file to be transferred.
 
Compression

Fast transfers

In order to achieve the fastest possible transfer times, TeamDrive compresses files when they are being transferred.
 
Instant loads

Local caching speeds subsequent access

After a file has been accessed, it is stored in the local cache. The next time that file is accessed, TeamDrive checks to see if the copy on the server has changed and, if not, uses the local copy. This allows files to be instantly loaded.
 
Collaboration

Sharing

TeamDrive allows files to be easily shared with others for collaborative work. File access rights can be assigned so that others can be given read-only or read-write access.

Locking

Files can be locked while they are being edited to prevent anyone else from changing the file at that exact time. This avoids the problems that occur when two or more users try to edit a file simultaneously and all but one of the users loses his or her changes. When a file is locked, other users can still view the document but not edit it.
 
Supports different servers

WebDAV, FTP, HTTP, XML, SOAP, CGI, ASP and proprietary server supportTeamDrive has a very flexible protocol layer that allows it to support many different types of servers. In addition to supporting standard WebDAV and FTP servers, TeamDrive can also support servers based on HTTP, XML, SOAP, CGI or ASP. This means that TeamDrive can be customized to support most servers without any server changes.
 
Easy installation

Small download 

The TeamDrive setup program is less than 1Mb so it can be downloaded quickly even with slow dial-up connections.

Quick install

TeamDrive has its own custom installation program that can be run directly from the web and does not require the user to answer a number of potentially confusing questions.

Pre-configured

For any of the co-branded or private label versions, TeamDrive comes pre-configured with the proper server information. All users need to do is to enter their usernames and passwords, then click Connect.
 
Co-branding, private label, customization

Co-branding

The first level of customization is co-branding. TeamDrive will be provided as pre-configured, with the service information including server location, type and custom graphic included. Users just need to enter their username and password, then click Connect.

Private label

TeamDrive private-labeling provides further customization. In this version of the product, even the name "TeamDrive" can be changed. Private labeling may include not only the users' screens but the program name, installation package and help pages as well.

Customized

Many partners would like to add additional features to TeamDrive that make the product work more effectively in their computing environment. This might include such items as support of different servers, backups, Explorer right-click choices to access print or fax services, screen changes, etc. TeamStream can provide these customizations and has dedicated staff that can address such requirements.
