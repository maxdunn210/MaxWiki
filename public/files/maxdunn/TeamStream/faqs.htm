How can I setup TeamDrive Client to work with a WebDAV server that is not on the default list?

First create a new service by clicking on Service button and selecting New Service menu item. 

In the Server field specify the server name, like www.mywebdav.com and then select WebDAV from the Protocol combobox.

Some WebDAV servers require a specially-formatted path to access the user's files. Enter the path in the Server Directory field. If the path should contain the username you may use #U notation and it will be replaced by TeamDrive with the actual username.

If your username is myname and the URL to access your files is www.mywebdav.com/~myname, then you need to put ~#U as your server directory path.

If you experience any problems connecting or working with a WebDAV server please contact us at info@teamstream.com.

I have ZoneAlarm installed on my computer and TeamDrive Client does not work. What should I do?

If you are using ZoneAlarm you will need to make few extra configuration steps to let TeamDrive work with this firewall. 

Normally ZoneAlarm blocks the low-level access to the remote servers. To let TeamDrive access a server you should add it to the list of permitted servers in ZoneAlarm. To do this, follow these steps:

  1. Open main window of ZoneAlarm and select Security tab.
  2. Click the Advanced button. ZoneAlarm Pro users: select LocalZone Contents tab.
  3. Click the Add button and select Host/Site.
  4. Enter a description and the host name of the server you want to access with TeamDrive. For example, if you wanted to access i-drive you would enter www.idrive.com as a host name.

To enable access to a server on your LAN you may select IP Address option and enter the IP address, for example 192.168.0.1.

How do I edit the un-rendered source code on Zope?

Start Zope with the "-W 8081" option (or add "WEBDAV_SOURCE_PORT=8081" to the z2.py file) and then put "8081" in the TeamDrive "Port" field.  Here is a more detailed explanation.

  1. If starting as a program on Windows, edit the "Start.bat" file in the "C:\Program Files\Zope" directory. (Make sure it is a capital W.) It now looks like this:

    "C:\Program Files\Zope\bin\python.exe" "C:\Program Files\Zope\z2.py" -D -W 8081 %1 %2 %3 %4 %5 %6 %7 %8 %9

    - or - 
  2. To the "z2.py" file, modify the line to read:

    WEBDAV_SOURCE_PORT=8081

  3. Restart Zope.
  4. In TeamDrive, select the Zope service on the Connect tab and then select Service, Edit. Next click the Advanced button and enter 8081 in the "Port" field.

That is it! Now, when you connect to Zope, you will be able to edit the un-rendered code.

What are the differences in the Sharing Levels?

Sharing Levels allows you to optimize the performance of TeamDrive with how much you will use it for sharing. If you are not sharing files with anyone, set the Sharing Level to None. If files are shared and changed frequently, change the Sharing level to Heavy. Here are the details on the different sharing  levels:

Sharing Level Cache Dir Refresh Time Auto Locks Check before open
None Y 15 min    
Light Y 10 min    
Medium   5 min Y  
Heavy   10 sec Y Y

Cache Dir: When a program requests a directory listing that is currently available in the local cache, the local listing is returned to the program. A directory listing request is subsequently sent to the server and when it is returned, a Windows call is made to indicate the directory listing has changed, which will cause Windows Explorer and some other programs to update their directory listing.

Refresh Time: Windows makes lots of requests for directory listings. It is not practical to send all of these directory requests to the server and wait for a reply. Instead, TD caches directory listings and returns the same listing. If Windows requests a directory listing and the refresh time has expired, TD will return the cached listing but then get a new server directory listing.

Auto Locks: When a file is opened, TD will automatically lock it first. When the program that opened the file closes, TD will automatically unlock it. Auto locks are set for 15 minutes and then extended when the time falls below 10 minutes. If a file has been manually locked for a period longer than 15 minutes, it will not be automatically unlocked.

Check before open: When this option is active, before a file is opened, TD will check to see if a newer version exists. This insures that even if with cached directory listings, any open file request will get the most current copy of the file.
